搭建属于你自己专属的"微屁恩"(ss-panel-v3+ss后端)

  • centos
  • linux
  • nginx
  • python
  • php
1.5k 次阅读  ·  读完需要 28 分钟
7
  1. 简介
    如果没有接触过ss-panel和ss的人会问，这是什么鸟？其实很简单，迫于伟大祖国坚固的城墙的压力，无奈被这股压力压得喘不过气（气的要死）。国内百度帖子坑人指数真是高的离谱，作为搞运维的我，真的是无法解决好多问题，只能求助国外的粑粑，无奈城墙太坚固，之前买的”微屁恩“都被下架或者无法使用了。所以只能利用国外的VPS搭建自己的"微屁恩"服务器，打造自己专属的才是可靠的。
    简单来讲SS-Panel就是Shadowsocks的用户管理工具或者是平台，Shadowsocks当然就是"微屁恩" Server和Client的简称喽！SS-Panel是一套功能齐全的shadowsocks用户面板程序,SS-Panel(V3)魔改版是热心网友基于V3再次开发出来的版本，他在原来的基础上做了大量的改进和强化，使得Shadowsocks的多用户管理面板更加的方便与强大。安装这款程序可以实现用户注册、用户管理、流量限制、签到、添加节点、充值购买、充值返利等诸多功能。搭建完毕后完全可以使用这款程序在线销售shadowsocks。但是我不建议在线销售，买个简单的VPS完全可以满足3-5个人的正常使用，如果想看美剧或者小电影，那还是自己独享或者买个大点的VPS吧。
  2. 安装步骤
    安装步骤配置步骤大体分为以下几点，与目录结构差不多，供读者大体了解具体的流程。
    • 准备工作
    • 安装配置LNMP环境
    • 创建虚拟主机和SS-Panel(V3)代码部署
    • 数据库配置和导库
    • 配置config.php
    • 账号管理及同步用户
    • 安装配置Shadowsocks后端
    • SS-Panel面板添加Shadowsocks后端节点
    • Shadowsocks PC端软件及配置
    • Shadowsocks APP端软件和配置

  3. 准备工作

       挑选一款适合你的VPS，网上有很多购买VPS的教程，大家可以百度一下，由于我用的VPS比较少，所以请大家自行考虑。

  4. 安装配置LNMP环境
    下载LNMP一键安装包

    yum -y install wget
cd /opt && wget -c http://soft.vpser.net/lnmp/lnmp1.2-full.tar.gz && tar zxf lnmp1.2-full.tar.gz && cd lnmp1.2-full
./install.sh

    选择安装数据库版本，默认是5.5，我这里选择默认，5.5比较省资源，如果你买的VPS是最便宜那个，建议你安装5.5

    输入一个Mysql Root用户的密码，这个一定要记住，之后导库会用到，不然你会有点麻烦哦，如果你是DBA，请忽略我的警告。

    输入完Mysql Root密码后回车，到达是否开启数据库引擎选项，默认不开启，建议也是不开启，省资源
    选择PHP版本，默认是5.6，是否开启内存优化，默认不开启，默认回车即可
    然后按任意键开始漫长的安装过程。。。这个过程会很慢，我的大概执行了40分钟，不要中断ssh连接哦！否则会中断安装的。

    当你看到类似于下图的时候，恭喜你，成功了！

  5. 创建虚拟主机和SS-Panel(V3)代码部署
    创建虚拟主机

    lnmp vhost add

    创建成功

    Nginx vhost配置
    使用vi命令打开一的Nginx站点的虚拟主机配置文件

    vi /usr/local/nginx/conf/vhost/你的域名.conf

    添加下面的这段代码

    location / {
                        try_files $uri $uri/ /index.php$is_args$args;
                }

    添加成功后的效果

    接着往上看到root那一行在后面添加一个public如图

    修改监听端口

    SS-Panel(V3)代码部署
    下载程序代码：

    cd /home/wwwroot/你的站点域名
yum install git -y
git clone https://github.com/glzjin/ss-panel-v3-mod.git tmp -b new_master && mv tmp/.git . && rm -rf tmp && git reset --hard
chown -R root:root *
chmod -R 777 *
chown -R www:www storage

    修改网站文件属性

    [root@VM_0_2_centos vpn.test.com]# chattr -i .user.ini
[root@VM_0_2_centos vpn.test.com]# mv .user.ini public
[root@VM_0_2_centos vpn.test.com]# cd public
[root@VM_0_2_centos public]# chattr +i .user.ini

    重启Nginx服务

    [root@VM_0_2_centos public]#service nginx restart

    查看Nginx服务进程

    [root@VM_0_2_centos public]# ps -ef | grep nginx

  6. 数据库配置和导库
    将sql文件复制到upload目录

    cp /home/wwwroot/你的站点域名/sql/glzjin_all.sql /home/wwwroot/default/phpmyadmin/upload

    现在我们通过浏览器打开我们的phpmyadmin：http://你VPS的IP/phpmyadmin/ 这里的IP地址是公网IP地址哦，不是内网地址，否则你是打不开的输入安装LNMP过程中的Mysql root用户的密码进行登录

    创建数据库管理用户，并创建一个同名的数据库
    这里需要注意Host name选项务必填%，表示任意主机可以连接数据库，如果你对数据库比较熟悉，也可以做远程连接控制

    接着我们将之前的本地glzjin_all.sql导入到新建的数据库中

  7. 配置config.php
    接下来我们修改ss-panel-v3-mod的相关配置

    cd /home/wwwroot/你站点的域名
php composer.phar install
cp config/.config.php.example config/.config.php
nano config/.config.php

    使用composer安装拓展包，这里会出现很多错误问题，可以忽略掉

    如果没有nano命令，请使用yum进行安装

    yum -y install nano

    使用nano命令打开config.php文件后，可以清晰的看到配置文件的含义，主要修改数据库连接信息

    填写好之后，使用Ctrl+X组合键退出nano编辑 模式，nano会提醒你是否保存配置，按Y键进行保存。之后会提示你是否重命名文件，直接回车即可。

  8. 账号管理及同步用户
    创建ss-panel-v3-mod的管理账户

    php -n xcat createAdmin

    输入邮箱和密码创建管理用户

    用户同步

    php xcat syncusers

    安装Crontab

    yum -y install cronie

    使用Linux Crontab增加一些定时任务

    crontab -e

    增加如下定时任务

    30 22 * * * php /home/wwwroot/你的站点域名/xcat sendDiaryMail
*/1 * * * * php /home/wwwroot/你的站点域名/xcat synclogin
*/1 * * * * php /home/wwwroot/你的站点域名/xcat syncvpn
0 0 * * * php -n /home/wwwroot/你的站点域名/xcat dailyjob
*/1 * * * * php /home/wwwroot/你的站点域名/xcat checkjob    
*/1 * * * * php -n /home/wwwroot/你的站点域名/xcat syncnas

    到这里我们的ss-panel-v3-mod的前端面板就已经安装完成了

  9. 安装配置Shadowsocks后端
    接下来我们安装后端的ss（Shadowsocks简称）
    安装libsodium

    yum -y groupinstall "Development Tools"
wget https://github.com/jedisct1/libsodium/releases/download/1.0.10/libsodium-1.0.10.tar.gz
tar xf libsodium-1.0.10.tar.gz && cd libsodium-1.0.10
./configure && make -j2 && make install
echo /usr/local/lib > /etc/ld.so.conf.d/usr_local_lib.conf
ldconfig

    安装Shadowsocks后端程序

    cd /opt
git clone -b manyuser https://github.com/glzjin/shadowsocks.git
cd shadowsocks
yum -y install python-devel libffi-devel openssl-devel
pip install -r requirements.txt

    创建配置文件

    cp apiconfig.py userapiconfig.py
cp config.json user-config.json

    编辑配置文件

    vi userapiconfig.py

    启动调试模式的ss，调试模式非后台程序，如果ssh中断或者Ctrl+C均会中断服务。这里启动是为了观察程序启动中发生的错误，根据错误去处理问题

    python /opt/shadowsocks/server.py

    如果后面的安装均没有问题，建议使用后台模式启动ss，命令如下

    ./run.sh
  10. SS-Panel面板添加Shadowsocks后端节点
    接下来我们登录ss-panel-v3-mod面板，输入我们增加的邮箱和密码进行登录。使用浏览器输入IP:12306

    添加ss节点
    接下来我们添加我们部署好的ss节点，登录后点击页面左下角的“管理面板”==>“节点列表”==> "+"(这里需要注意是一定得点左下角的管理面板再点击节点列表，不然你是看不到添加节点按钮的)

    输入节点信息

    点击左下角的“用户中心”==> "节点列表" 查看节点添加的节点信息

  11. Shadowsocks PC端Client软件安装和配置
    PC客户端下载
    链接描述
    下载软件后，解压运行，在PC任务栏中会出现一个小飞机的图标，右键“服务器”==>“编辑服务器”，然后点击添加按钮，输入服务器地址，端口号和密码

    获取端口和密码
    打开后台ss-panel-mod的管理页面，点击左下角“管理面板”==>“用户列表”，这里会看到我们之前创建的管理用户信息，点击编辑按钮，就会看到我们的用户相关信息。

    保存配置信息即可，接下来，你就可以愉快的浏览网页了。
    服务器代理模式分为两种，一种是全局代理一种是PAC模式。
    全局模式：

       简单来讲就是所有的流量都会经过ss，这样可以保证你访问的网站一定可以通过代理，但是可能代开国内网站会比较慢，并且最最最重要的是，他走流量啊

    PAC模式：

       PAC模式类似于网站白名单，所有在白名单中的网站才会走ss，不在白名单中的就不会走ss，这样一来可以省掉很多流量，但是，如果你访问的网站不在白名单内，就会打不开，比如GitHub，那么你就得手动切换代理模式或者将github的域名加入到PAC模式的白名单文件中。

    详细的大家可以右键小灰机，看一下配置选项，全中文，无障碍。。。

  12. Shadowsocks APP端软件和配置
    安装：
    由于国内对“微屁恩”各种打压，所以迫于政策，你已经不能在APP Store下载小火箭了，所以有两种方法可以安装客户端
    方法一：

       使用美国或者非大陆国家注册的AppleID登录你的APP Store，搜索Shadowsocket安装即可

    方法二：

       手动下载Shadowsocket Apa文件，使用类似于PP助手的软件连接电脑安装，我上传到了Github上，链接如下

    链接描述
    配置：
    在电脑PC端，右键小灰机，“服务器” ==> “分享服务器配置”
    打开手机，打开客户端Shadowsocket，点击左上角的扫一扫，扫描PC端生成的二维码，所有在电脑上的配置，完全会同步到手机，灰常好用
